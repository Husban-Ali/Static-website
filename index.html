1. Hard-coded Secrets in Frontend

Storing API keys or sensitive information directly in frontend code exposes them to everyone viewing the source.

const SECRET_KEY = "12345-ABCDE";
Risk: Unauthorized access and data leaks.

2. Insecure Contact Form

Data sent over unencrypted HTTP or without validation can be intercepted or manipulated.

Use HTTPS and validate inputs on server.

3. Unsafe DOM Manipulation

Using innerHTML to display user input can lead to Cross-Site Scripting (XSS) attacks.

Safe preview uses textContent instead of innerHTML.

4. Third-Party Script Risks

Including external scripts from untrusted sources can introduce malicious code.

<script src="https://cdn.example.com/script.js"></script>
Use Subresource Integrity (SRI) and Content Security Policy (CSP).

Summary of Weaknesses

  • Hard-coded secrets: Easily accessible in frontend code, risk of unauthorized access.
  • Insecure contact form: Data can be intercepted if sent over HTTP without validation.
  • Unsafe DOM manipulation: Using innerHTML can lead to XSS attacks.
  • Third-party scripts: External scripts from untrusted sources may include malicious code.
